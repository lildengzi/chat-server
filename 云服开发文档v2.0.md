# Go 语言实现聊天室 MVP（最小可行性产品）

> 核心目标：在保留 Docker + PostgreSQL + Redis + Nginx 架构的基础上，用 **Go** 替代 C++，扔掉安全性、分区、加密等复杂设计，实现一个**可横向扩展、能跑满 4C2G 服务器 70% 负载**的极简聊天室。  
> 附带**配套 Web 客户端**（HTML+JS），一键 docker-compose 启动即可体验。

---

## 一、简化后的架构

![简化架构](https://via.placeholder.com/800x300?text=Web+Client+-->+Nginx+-->+Go+Server(x2)+-->+PostgreSQL+&+Redis)

- **Nginx**：反向代理 + WebSocket 升级 + 简单轮询负载均衡  
- **Go 服务（2 个实例）**：处理 WebSocket、用户管理、消息路由  
- **PostgreSQL**：存用户、好友关系、离线消息  
- **Redis**：在线用户状态（`online:{userId}` 存储实例标识） + Pub/Sub（跨实例广播消息）

**去掉的功能**（未来可加）：
- 朋友圈、点赞、评论  
- 消息加密、身份证加密  
- 分区表、pg_partman  
- 复杂的好友申请流程（简化为直接加好友）  
- 离线消息确认机制（改为“拉取后删除”）  
- 雪花算法 ID（使用自增 int64 + 数据库序列）

**保留的核心**：
- 用户注册/登录  
- 点对点私聊 + 离线消息  
- 在线用户列表  
- 添加/删除好友  
- 跨实例消息广播

## 二、数据库设计（超级简化版）

```sql
-- 初始化脚本 init.sql
CREATE DATABASE chatdb;
\c chatdb;

CREATE TABLE users (
    user_id   BIGSERIAL PRIMARY KEY,
    username  VARCHAR(64) UNIQUE NOT NULL,
    password  TEXT NOT NULL,            -- 明文（省事），实际可加 md5
    avatar    TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE friends (
    id         BIGSERIAL PRIMARY KEY,
    user_id    BIGINT NOT NULL,
    friend_id  BIGINT NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(user_id, friend_id)
);

CREATE TABLE offline_messages (
    msg_id      BIGSERIAL PRIMARY KEY,
    to_user_id  BIGINT NOT NULL,
    from_user_id BIGINT NOT NULL,
    content     TEXT NOT NULL,
    created_at  TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_offline_to ON offline_messages(to_user_id);
```

**说明**：
- 密码明文存储（不安全，但学习阶段无妨）  
- 好友关系单向（双人关系实际需要两条记录，或查询时处理，为简化我们查询时 `WHERE user_id=$1 OR friend_id=$1`）  
- 离线消息拉取后立即删除（不保留复核机制）

### 3.3 在线用户管理（Hub）

```go
// hub.go
type Client struct {
    userID   int64
    username string
    conn     *websocket.Conn
    send     chan []byte
}

type Hub struct {
    clients    map[int64]*Client      // userID -> Client
    register   chan *Client
    unregister chan int64  // userID
    broadcast  chan *Message          // 跨实例广播使用
}

var hub = &Hub{
    clients:    make(map[int64]*Client),
    register:   make(chan *Client),
    unregister: make(chan int64),
    broadcast:  make(chan *Message, 256),
}

func (h *Hub) Run() {
    for {
        select {
        case c := <-h.register:
            h.clients[c.userID] = c
            // 记录到 Redis: online:userId = instanceId
            rdb.Set(ctx, fmt.Sprintf("online:%d", c.userID), instanceID, 120*time.Second)
        case uid := <-h.unregister:
            if c, ok := h.clients[uid]; ok {
                delete(h.clients, uid)
                close(c.send)
                rdb.Del(ctx, fmt.Sprintf("online:%d", uid))
            }
        case msg := <-h.broadcast:
            // 发给本实例内的目标用户
            if c, ok := h.clients[msg.ToUserID]; ok {
                c.send <- msg.ToBytes()
            }
        }
    }
}
```

### 3.4 WebSocket 处理流程

```go
// handlers.go
func serveWs(w http.ResponseWriter, r *http.Request) {
    // 1. 从 query 获取 user_id（登录后前端携带）
    userID, _ := strconv.ParseInt(r.URL.Query().Get("user_id"), 10, 64)
    // 2. 验证用户存在（查数据库）
    var user User
    err := dbPool.QueryRow(ctx, "SELECT user_id, username FROM users WHERE user_id=$1", userID).Scan(&user.UserID, &user.Username)
    if err != nil {
        w.WriteHeader(401)
        return
    }

    conn, err := upgrader.Upgrade(w, r, nil)
    if err != nil { return }

    client := &Client{
        userID:   user.UserID,
        username: user.Username,
        conn:     conn,
        send:     make(chan []byte, 256),
    }
    hub.register <- client

    // 启动读协程和写协程
    go writePump(client)
    go readPump(client)
}
```

**读消息处理**（`readPump`）：

```go
func readPump(c *Client) {
    defer func() {
        hub.unregister <- c.userID
        c.conn.Close()
    }()
    for {
        var msg Message
        err := c.conn.ReadJSON(&msg)
        if err != nil { break }

        switch msg.Type {
        case "chat":
            // 私聊消息
            go handleChat(c, &msg)
        case "get_online_list":
            go sendOnlineList(c)
        case "add_friend":
            go addFriend(c, msg.ToUserID)
        case "get_offline":
            go sendOfflineMessages(c)
        }
    }
}
```

**发送聊天消息（核心：跨实例广播）**：

```go
func handleChat(from *Client, msg *Message) {
    // 1. 存储到离线消息表（无论在线与否，简化：总是存，但下面会处理实时发送，上线后再拉取会重复？）
    //    优化：先判断在线，若在线则实时发，不存离线；若不在线则存离线
    toUserID := msg.ToUserID
    content := msg.Content

    // 检查对方是否在线（查 Redis）
    targetInst, err := rdb.Get(ctx, fmt.Sprintf("online:%d", toUserID)).Result()
    if err == nil && targetInst != "" {
        // 在线，通过 Redis Pub/Sub 发到对方实例
        pubMsg := Message{
            Type:       "chat",
            FromUserID: from.userID,
            Content:    content,
        }
        rdb.Publish(ctx, "chat_broadcast", mustJson(pubMsg))
        return
    }

    // 不在线，存离线
    _, err = dbPool.Exec(ctx,
        "INSERT INTO offline_messages (to_user_id, from_user_id, content) VALUES ($1,$2,$3)",
        toUserID, from.userID, content)
    if err != nil {
        log.Println("save offline msg failed:", err)
    }
}
```

**每个实例订阅 Redis 频道**：

```go
// main.go 启动时
go func() {
    pubsub := rdb.Subscribe(ctx, "chat_broadcast")
    defer pubsub.Close()
    for msg := range pubsub.Channel() {
        var chatMsg Message
        json.Unmarshal([]byte(msg.Payload), &chatMsg)
        // 发送给本实例内的目标用户（如果在本地）
        if c, ok := hub.clients[chatMsg.ToUserID]; ok {
            c.send <- mustJson(chatMsg)
        }
    }
}()
```

**用户上线拉取离线消息**：

```go
func sendOfflineMessages(c *Client) {
    rows, _ := dbPool.Query(ctx,
        "SELECT msg_id, from_user_id, content FROM offline_messages WHERE to_user_id=$1 ORDER BY created_at",
        c.userID)
    defer rows.Close()

    var msgs []Message
    for rows.Next() {
        var msg Message
        rows.Scan(&msg.MsgID, &msg.FromUserID, &msg.Content)
        msg.Type = "chat"
        msgs = append(msgs, msg)
    }
    // 拉取后立即删除（简化）
    dbPool.Exec(ctx, "DELETE FROM offline_messages WHERE to_user_id=$1", c.userID)

    c.send <- mustJson(Message{Type: "offline_list", Data: msgs})
}
```

### 3.5 登录/注册 HTTP API（便于前端获取 user_id）

```go
func loginHandler(w http.ResponseWriter, r *http.Request) {
    var req struct{ Username, Password string }
    json.NewDecoder(r.Body).Decode(&req)

    var user User
    err := dbPool.QueryRow(ctx,
        "SELECT user_id, username FROM users WHERE username=$1 AND password=$2",
        req.Username, req.Password).Scan(&user.UserID, &user.Username)
    if err != nil {
        w.WriteHeader(401)
        return
    }
    json.NewEncoder(w).Encode(user)
}

func registerHandler(w http.ResponseWriter, r *http.Request) {
    var req struct{ Username, Password string }
    json.NewDecoder(r.Body).Decode(&req)
    var userID int64
    err := dbPool.QueryRow(ctx,
        "INSERT INTO users (username, password) VALUES ($1,$2) RETURNING user_id",
        req.Username, req.Password).Scan(&userID)
    if err != nil {
        w.WriteHeader(400)
        return
    }
    json.NewEncoder(w).Encode(map[string]int64{"user_id": userID})
}
```

            let pwd = document.getElementById('login-password').value;

                method: 'POST',

## 七、性能评估（4C2G 极限 70% 负载）

**理论负载**（简化估算）：
- Go 每万连接内存约 200MB（goroutine + 缓冲区），2G 内存可支撑 ~5000 活跃 WebSocket。  
- CPU 瓶颈在于消息广播（所有实例收到所有消息），4 核机器每个实例分配 0.7 核，可处理约 2000 msg/s。  
- 达到 70% 负载时：约 3500 在线用户，每秒 1500 条消息。  
- 若需更高性能，可改用**一致性哈希**减少广播，但 MVP 已够用。

**压力测试建议**：使用 `gorr` 或 `websocket-bench` 模拟 2000 连接，观察 CPU < 70%。

## 八、总结：从痛苦到快乐

| 原 C++ 方案 | Go MVP 方案 |
|------------|-------------|
| 手写 Asio 回调地狱 | 天然的 goroutine 并发 |
| 异步连接池、复杂内存管理 | 标准库 database/sql + 连接池 |
| 需要手动管理 WebSocket 分片 | gorilla/websocket 全自动 |
| 数百行 cmake / vcpkg | `go mod tidy` 一行搞定 |
| 部署需处理二进制兼容 | 静态编译，丢进容器就行 |

**后续可扩展方向**（让 MVP 慢慢长大）：
- 加入 JWT 认证（安全性）  
- 改用 Redis 一致性哈希路由（减少广播）  
- 存储聊天历史到 PostgreSQL（单表即可）  
- 增加群聊功能  

你现在拥有一个**可运行、可扩展、容易理解**的聊天室骨架。把精力放在理解消息流转和分布式协调上，而不是和 C++ 内存搏斗。祝你 coding 愉快！
